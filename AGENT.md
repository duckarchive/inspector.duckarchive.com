## Project Overview

**inspector.duckarchive.com** — a web app for exploring Ukrainian genealogy archives. It presents the archive hierarchy (Archive → Fund → Description → Case) and tracks online availability of digitized materials across resources (e.g. FamilySearch DGS). Part of the Duck Archive ecosystem (duckarchive.com).

## Tech Stack

- **Framework:** Next.js 15 (App Router, Turbopack in dev), React 19, TypeScript 5.8
- **UI:** HeroUI v2 (per-component packages `@heroui/*`), Tailwind CSS 3, tailwind-variants, Framer Motion, react-icons
- **Tables/Charts:** AG Grid (`ag-grid-react`, marked as `serverExternalPackages` in next.config.js), AG Charts
- **Maps:** Leaflet / react-leaflet + `@duckarchive/map`
- **Data:** PostgreSQL via Prisma 7 (`@prisma/client` + `@prisma/adapter-pg`), Zod 4 (schemas generated by `prisma-zod-generator`)
- **Auth:** NextAuth v4 with Google OAuth (`app/api/auth/[...nextauth]/route.ts`); access token kept in JWT/session. Server-side user lookup against external Duck API via `lib/user.ts` (`Authorization: Bearer` header → `DUCK_API_URL/api/user`)
- **i18n:** next-intl. Locales: `uk` (default, no URL prefix), `en`, `pl`, `cz`, `ro`, `es`, `it`. Config in `i18n/`, translations in `messages/*.json`
- **Data fetching (client):** SWR with fetchers from `lib/api.ts`
- **Shared org packages (installed from GitHub):** `@duckarchive/framework`, `@duckarchive/map`, `@duckarchive/prisma`
- **Package manager:** pnpm (see `pnpm.onlyBuiltDependencies` in package.json)

## Commands

```bash
pnpm dev              # dev server (next dev --turbo)
pnpm build            # production build
pnpm start            # serve production build
pnpm lint             # eslint --fix on .ts/.tsx
pnpm clear            # rm -rf .next
pnpm prisma:generate  # regenerate Prisma client + Zod schemas into generated/prisma/ (also runs on postinstall)
pnpm notify:tg        # scripts/notification.ts — send Telegram notification (reads _notification.json)
pnpm cache:purge      # scripts/purge-cache.ts — purge Cloudflare cache
```

There is no test suite in this repo.

## Prisma Specifics (important)

- The schema does **not** live in this repo. `prisma.config.ts` points to `node_modules/@duckarchive/prisma/prisma/inspector/schema.prisma` (shared `@duckarchive/prisma` package).
- Generated client goes to `generated/prisma/client`, generated Zod schemas to `generated/prisma/zod`. Import via the `@generated/*` path alias (e.g. `import { PrismaClient } from "@generated/prisma/client/client"`).
- DB access goes through the singleton in `lib/db.ts` (PrismaClient with `PrismaPg` adapter).
- To change the DB schema, change the `@duckarchive/prisma` package, then reinstall/regenerate here.

## Conventions

- **Path aliases:** `@/*` → repo root, `@generated/*` → `generated/`.
- **API route pattern:** each `app/api/...` level pairs `route.ts` (HTTP handler) with `data.ts` (data-access logic). Follow this split when adding endpoints.
- **Archive codes in URLs** are Cyrillic-capable — see `useCyrillicParams` hook for decoding route params.
- **Pages:** every archive-hierarchy page level has a `loading.tsx`. `[archive-code]` also has `opengraph-image.tsx`.
- **Linting:** flat ESLint config (`eslint.config.mjs`) with prettier, react-hooks, unused-imports, jsx-a11y. Run `pnpm lint` before committing (it auto-fixes).
- **Localization:** any user-facing string must be added to all 7 files in `messages/`. Default locale `uk` is served without a URL prefix (`localePrefix: "as-needed"`).
- **CORS:** API is consumed cross-origin by duckarchive.com and a Chrome extension — allowlist lives in `middleware.ts`.

| `DATABASE_URL` | Prisma (lib/db.ts, prisma.config.ts) |

Scripts load env via `dotenv` (`tsx -r dotenv/config`).

## Gotchas

- `generated/` is produced by `pnpm prisma:generate` — never hand-edit; regenerate after dependency updates.
- The Prisma schema is inside `node_modules` (from `@duckarchive/prisma`), so `pnpm install` order matters: `postinstall` runs generation automatically.
- `ag-grid-community` must stay in `serverExternalPackages` (next.config.js) or the build breaks.
- Build uses `staticPageGenerationTimeout: 300` and `staticGenerationMaxConcurrency: 2` — static generation of archive pages is heavy; be careful adding more `generateStaticParams`.
- README.md is the stale Next.js/NextUI template readme — don't trust it for project info; trust this file.
